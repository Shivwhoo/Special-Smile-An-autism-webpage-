SPECIAL SMILE CENTER - FULL CODESPACE CONTEXT SUMMARY
======================================================

This document outlines the complete system architecture, folder structure, API backend mappings, database schemas, and integrated service configurations that form the Special Smile Center application ecosystem.

1. System Overview
------------------
Special Smile is a full-stack web application designed to support parent-clinician onboarding and session scheduling.
- Frontend: Single Page Application (SPA) built with React, Redux Toolkit (state management), framer-motion (animations), and TailwindCSS (styling), powered by Vite.
- Backend: RESTful API server powered by Node.js, Express, and Mongoose (MongoDB ODM).
- Third-Party Integrations:
  * Google Calendar API: Automated headless server-to-server appointment event synchronization.
  * Resend Email API: Automated transaction validation and support notification emails.

3. Core Integrations & Workflows
--------------------------------

A. Booking & Availability Engine:
1. When a parent clicks on a date in the custom calendar grid ("BookAppointment.jsx"), the client queries "GET /api/appointments/booked-slots?date=YYYY-MM-DD".
2. The controller ("appointmentController.js") fetches confirmed slots on that specific day and returns them to the frontend.
3. The calendar disables occupied slots on the interface.
4. On submission, the backend checks for conflicts (double-booking protection) and inserts a "confirmed" entry.

B. Headless Google Calendar Sync:
- Authentication: The backend authenticates server-to-server using a Google Cloud Service Account JWT.
- API Call: Utilizing the "googleapis" SDK, it calls "calendar.events.insert" using the correct "requestBody" parameters to prevent SDK errors.
- Sync: Automatically schedules the appointment in the clinic's administrative calendar (within the "Asia/Kolkata" timezone) and assigns the resulting Google Event ID to Mongoose.
- Fallback: Bypasses sync gracefully with console logs if service keys are unconfigured, ensuring the local database is still updated successfully.

C. Transactional Emails:
- Instantly sends beautifully styled HTML confirmations to:
  * The parent's registered email.
  * The administrative clinic inbox ("jaylovelypriya@gmail.com").
